整个界面设计采用了“**左侧导航，右侧详情**”的经典布局，并在顶部设有全局控制。设计语言简洁，通过颜色和简短的标签来传达状态。

---

### 1. 顶部控制栏 (Header)

这是系统的“总指挥部”，用于控制全局状态。

* **Session Start (会话开始)**: 显示本次监控服务启动的时间。
* **🗑 Reset (重置)**: 点击后清空**当前内存中**的统计计数（不会删除数据库里的历史日志）。当您想重新观察一轮新的数据积累时使用。
* **Run / Pause (运行/暂停)**: 全局红绿灯。
* **Run (绿)**: 系统正常调度任务。
* **Pause (黄)**: 系统暂停派发新任务（正在运行的任务会继续完成）。


* **FLOW 按钮**: 显示流量调度器的实时状态。
* **RUN**: 当前并发数。
* **Q**: 排队等待的任务数。
* *点击该按钮可以打开弹窗，查看具体的队列详情。*


* **⬇ Report**: 下载全局统计报表（CSV格式）。
* **Total Results**: 本次会话累计处理的任务总数。

---

### 2. 左侧资源管理器 (Explorer)

这里列出了所有的爬虫组（Group）。

* **树状结构**: 点击 `+` / `-` 按钮可以展开或折叠所有文件夹。
* **状态指示灯 (圆形图标)**: 每个爬虫组左侧都有一个小圆点，代表该组的健康状况：
* 🔵 **蓝色闪烁**: **运行中 (Running)**。该组正在进行爬取任务。
* 🟢 **绿色**: **健康 (Success)**。最近一次检查（锚点/列表页）是成功的。
* 🔴 **红色**: **故障 (Fail)**。列表页访问失败或解析错误，需要关注。
* ⚫ **灰色**: **闲置/等待**。


* **数字徽标**:
* `R123`: 代表当前轮次 ID (Round ID)。
* 数字 (如 `50`): 代表该组在本次会话中已产出的结果数量。

### 3. 中间主面板 (Main Content)

当您在左侧点击某个组时，右侧会显示该组的详细信息。

#### A. 顶部详情卡片 (Group Details)

这一栏展示了该爬虫组最关键的三个维度的实时快照：

1. **Anchor / List (锚点状态)**:
* 监控入口页（如列表页）的状态。
* 显示 HTTP 状态码（如 `HTTP 200` 为绿，`HTTP 404/500` 为红）。
* 显示系统给出的状态消息（如 "System Normal" 或报错信息）。


2. **Round (轮次进度)**:
* **进度条**: 显示当前这一轮抓取的完成度。
* **统计**: `OK` (成功数) / `Err` (失败数) / `进度/预期总数`。


3. **Perf (性能)**:
* **Avg**: 平均响应时间（秒）。
* **Run**: 当前正在进行的并发请求数。



#### B. 视图切换 (View Modes)

位于详情卡片下方，提供三种查看数据的视角：

1. **Live (实时流)**: 显示最近发生的日志，适合盯着看系统在干什么。
2. **History (历史回溯)**:
* 会出现时间选择器（开始时间 - 结束时间）。
* 点击 `Query` 查询数据库中的历史记录。


3. **Chart (趋势图)**:
* 展示任务量的柱状图。
* **绿色部分**: 成功/总量。
* **红色塔尖**: 失败数量（如果有红色，说明该时间段有报错）。
* 下拉菜单可选择时间范围（最近1小时、6小时、24小时）。

### 4. 列表符号与状态代码说明 (Status Glossary)

在日志列表中，您会看到各种缩写标签和颜色，它们的含义如下：

| 标签 (Tag) | 颜色 | 含义 (Meaning) | 说明 |
| --- | --- | --- | --- |
| **PEND** | 灰色 | Pending (等待) | 任务已发现，正在排队等待执行。 |
| **RUN** | 🔵 蓝底 | Running (运行中) | 正在下载或解析，尚未结束。 |
| **OK** | 🟢 绿底 | Success (成功) | 任务顺利完成，数据已提取。 |
| **TMP** | 🟡 黄底 | Temp Fail (临时失败) | 网络超时或暂时不可用，**系统会自动重试**。 |
| **PRM** | 🔴 红底 | Perm Fail (永久失败) | 404、解析崩溃等严重错误，**不会重试**。 |
| **SKIP** | 灰色 | Skipped (跳过) | 根据规则（如去重、过滤器）跳过了该任务。 |
| **STOP** | 灰色 | Stopped (停止) | 人工停止或系统关闭导致的中断。 |
| **CACHED** | 🟢 绿底 | Cached (缓存) | 命中了缓存，直接视为成功。 |
| **IGN** | 灰色 | Ignored (忽略) | 系统内部处理标记，通常不计入统计。 |

**其他列表字段：**

* **Time**: 事件发生的时间。
* **Code**: HTTP 状态码（如 200, 403, 404, 500）。**红色**代表 >= 400 的错误。
* **Dur(s)**: 耗时（秒）。

### 5. 常见操作流程

* **我想看某个爬虫是不是挂了？**
* 看左侧列表，找有没有🔴红点的组。
* 点击该组，看顶部的 "Anchor / List" 卡片是不是显示红色 HTTP 错误。


* **我想看最近有没有报错？**
* 点击左侧组，切换视图到 **Chart**。
* 看柱状图顶端有没有红色的条块。


* **我想查昨天某个时间段的日志？**
* 点击 **History** 视图。
* 选择昨天的时间范围，点击 Query。


* **系统好像卡住了？**
* 点击顶部的 **FLOW** 按钮。
* 如果 `Run` 很低但 `Q` (队列) 很高，可能是并发限制生效了。
* 如果 `Run` 和 `Q` 都不动，检查一下顶部的状态是不是被设为了 **Pause**。
